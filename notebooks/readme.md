### 🐣 Classic CV Approach (Baseline)

**Задача:** Подсчет уникальных цыплят, движущихся вниз в трех вертикальных колонках конвейера на видеозаписи, с визуализацией треков и сохранением результата.

#### ⚙️ Алгоритм работы

1.  **Предобработка кадра**:
    *   Видео открывается и читается покадрово.
    *   Верхняя часть кадра (60 пикселей) зануляется (закрашивается черным) для исключения ложных срабатываний от оборудования сверху.
    *   Кадр конвертируется в цветовое пространство **HSV**.

2.  **Сегментация (Цветовой фильтр)**:
    *   Применяется маска для выделения желтого цвета (`lower_yellow` – `upper_yellow`), соответствующего окрасу цыплят.
    *   Используются морфологические операции (`MORPH_OPEN`, `MORPH_CLOSE`) для удаления шумов и заполнения дыр в контурах.

3.  **Выделение объектов**:
    *   Находятся контуры (`findContours`).
    *   Фильтруются по площади: игнорируются объекты меньше **30,000 пикселей** (шум/мусор).
    *   Вычисляется центр масс `(cx, cy)` каждого объекта.

4.  **Зонирование (Логика колонок)**:
    *   Кадр жестко разделен на вертикальные полосы (колонки). Код проверяет попадание центра объекта (`cx`) в диапазоны:
        *   **Row 2** (левая рабочая зона)
        *   **Row 3** (центральная рабочая зона)
        *   **Row 4** (правая рабочая зона)
    *   Объекты за пределами этих зон игнорируются.

5.  **Трекинг и подсчет (Упрощенный)**:
    *   Используется простая логика сопоставления между кадрами:
        *   Если объект находится в той же зоне и его координата `Y` увеличилась (движение вниз/по конвейеру), он считается продолжением предыдущего трека.
        *   Если совпадения нет, создается **новый уникальный ID**.
    *   Уникальные ID сохраняются в множествах `seen_ids` для каждой зоны. Это гарантирует, что один и тот же цыпленок не будет посчитан дважды, даже если трек потерялся и создался заново (при условии, что он не появился "сверху" снова).

6.  **Визуализация и вывод**:
    *   На кадр наносятся границы зон, прямоугольники вокруг цыплят и их ID.
    *   Результат сохраняется в видеофайл `color_tracked.mp4`.
    *   Имя выходного файла переименовывается с добавлением общего количества подсчитанных объектов.
    *   В консоль выводится статистика: количество цыплят по каждой колонке и итого.

#### 💡 Ключевые особенности

*   **Не использует нейросети**: Работает исключительно на классическом компьютерном зрении (OpenCV + HSV), что очень быстро, но чувствительно к изменению освещения и цвета фона.
*   **Однонаправленный трекинг**: Учитывает движение только "вниз" (`prev_cy < cy`), что эффективно для конвейера.
*   **Жесткая геометрия**: Размеры колонок заданы константами под конкретное разрешение камеры (1624x1240).
