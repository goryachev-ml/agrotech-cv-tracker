End-to-end Computer Vision пайплайн для трекинга и подсчета цыплят на птицефабрике с точностью 99%. Разработан мной для внедрения на реальном производстве (ООО «Алгофьюжн»).

Система в реальном времени обрабатывает видеопотоки с камер над конвейерами, проводит сегментацию объектов, отслеживает их перемещение, отсеивает аномалии и отправляет аналитику в пользовательский интерфейс и Telegram-бот.

## 🚀 Ключевые особенности 
В данном репозитории представлена **моя реализация** пайплайна. В рамках работы над проектом я реализовал следующие задачи:

* **Инженерия данных и авторазметка:** Настроил пайплайн авторазметки датасета с использованием **SAM-2 / HQ-SAM**. Разработал стандарты разметки и сформировал эталонный датасет (LabelStudio, RoboFlow, VGG).
* **Обучение ML-моделей:** Обучил и оптимизировал сегментационные модели от YOLOv8n до **YOLO11x-seg**. 
* **Компьютерное зрение (CV):** 
  * Реализовал математическую логику фильтрации «фантомных» треков (трекинг-лайфтаймы, учет `MIN_TRACK_LENGTH`).
  * Внедрил HSV-сегментацию, морфологические операции и работу с контурами для сложных участков перекрытия.
  * Настроил вложенные эллиптические зоны трекинга.
  * Оптимизировал параметры **ByteTrack** под высокую плотность объектов на конвейере.
* **Интерфейс и интеграция:** Разработал многопоточный дашборд на **Gradio** (ранее Streamlit) для мониторинга 3-х камер одновременно и написал **Telegram-бота** (Aiogram 3) для запроса видеофрагментов с камер по триггерам.

## 🛠 Технологический стек
* **CV & ML:** Ultralytics (YOLO-seg), Segment Anything 2 (SAM-2), ByteTrack, OpenCV, PyTorch, NumPy.
* **Data Prep:** LabelStudio, RoboFlow, VGG Image Annotator.
* **Backend & UI:** Python, Gradio, Pandas, Aiogram, Asyncio.

## ⚙️ Архитектура решения (Пайплайн)
1. **Захват кадра:** Чтение RTSP/HTTP потока (до 15-30 FPS).
2. **Сегментация & Детекция:** YOLO извлекает маски объектов (`get_bbox_from_mask`). Для повышения точности используется попиксельное наложение цветовых фильтров.
3. **Трекинг (ByteTrack):** Каждому объекту присваивается ID. Алгоритм учитывает перекрытия (occlusion).
4. **Фильтрация аномалий:** Система ждет достижения объектом `MIN_TRACK_LENGTH` (например, 7 кадров), чтобы исключить шум, блики и "фантомные" детекции. Только после этого объект регистрируется в CSV-логере.
5. **Аналитика:** Статистика за минуту/час/день агрегируется с помощью `pandas` и выводится в UI.
